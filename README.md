(Software Testing)

---

## Giới thiệu học phần
Học phần **Kiểm thử phần mềm** cung cấp cho sinh viên các kiến thức nền tảng và kỹ năng thực hành liên quan đến việc đảm bảo chất lượng phần mềm. Môn học giúp sinh viên hiểu được vai trò của kiểm thử trong vòng đời phát triển phần mềm và cách phát hiện lỗi một cách có hệ thống.

---

## Mục tiêu học phần
Sau khi hoàn thành học phần, sinh viên có thể:
- Hiểu các khái niệm cơ bản về kiểm thử phần mềm
- Phân biệt các mức độ và loại hình kiểm thử
- Thiết kế và thực hiện các ca kiểm thử (test case)
- Áp dụng kiểm thử đơn vị (Unit Test) với JUnit
- Sử dụng công cụ hỗ trợ kiểm thử và quản lý lỗi

---

## Nội dung chính
- Tổng quan về kiểm thử phần mềm
- Kiểm thử hộp đen và kiểm thử hộp trắng
- Các mức độ kiểm thử: Unit Test, Integration Test, System Test
- Viết test case và kịch bản kiểm thử
- Kiểm thử tự động với JUnit
- Báo cáo lỗi và đánh giá chất lượng phần mềm

---

## Công cụ sử dụng
- Ngôn ngữ lập trình: Java
- Framework kiểm thử: JUnit
- IDE: IntelliJ IDEA / Eclipse
- Git & GitHub

---

## Mục đích của repository
Repository này được sử dụng để:
- Lưu trữ các bài tập thực hành của học phần
- Thực hành viết kiểm thử đơn vị
- Quản lý mã nguồn và lịch sử làm việc với GitHub
- Nộp bài và đánh giá kết quả học tập

---

## Ghi chú
Sinh viên cần:
- Thực hiện đầy đủ các bài tập thực hành
- Viết mã nguồn rõ ràng, dễ hiểu
- Viết test case đầy đủ cho các chức năng
- Tuân thủ quy định về commit và quản lý source code
